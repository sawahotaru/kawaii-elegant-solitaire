## 🔍 重大な不具合の修正記録 (デバッグ・フェーズ)

実機テスト中に発見された複数の複雑なバグに対する修正プログラムです。

### バグA: カードの操作ロック（一度動かすと反応しなくなる）
- **現象**: カードを他の列に移動させた直後、そのカードや新しく表になったカードがドラッグできなくなる。
- **原因**: Reactのコンポーネント再利用（Reconciliation）により、`useDraggable` の内部状態が古い場所の情報を保持し続けていた。
- **対策**: `key` 属性と `draggableId` をユニーク化。さらに、カードが表裏反転した瞬間にコンポーネントを強制再生成する「Flip-Stable Key」を導入し、完全なリセットを保証。

### バグB: カードの消失と移動スナップバック
- **現象**: 素早い操作をするとカードが消える、あるいは元の場所に戻される。
- **原因**: 状態更新時のシャローコピーによるオブジェクト参照の重複が、`dnd-kit` の追跡を混乱させていた。
- **対策**: ステートエンジン（`useGameState`）におけるすべての更新を **Deep Clone (深いコピー)** に刷新。データの独立性を確保。

### バグC: 特定の列がクリックできない（右から2列目）
- **現象**: 画面サイズによって、右から2列目のカードが掴めない。
- **原因**: 右隣の列の透明な余白が、重なり順の影響で左側の列を遮っていた。
- **対策**: Tableau列の `z-index` を反転（左側を高く設定）。左側の列が常に手前に来るようにし、クリックの遮断を解消。

### バグD: ドラッグ終了の失敗
- **現象**: ドラッグを離してもカードが指に付いたまま、あるいは不安定になる。
- **原因**: ドラッグ中に表示されるオーバーレイ要素がマウスイベントを遮断していた。
- **対策**: `DragOverlay` に `pointer-events-none` を適用し、背景のドロップ判定を確実に受け取れるように調整。

## 🔬 追加の検証と修正（解析フェーズ）

実コードによるロジック実行テスト（Node）と、ブラウザ自動化（Playwright）による実機テストを通じて、追加で2件の不具合を発見・修正しました。

### バグE: ヒントが実行不可能な手を提示する
- **現象**: `findValidMoves` が、上に他のカードが乗って「埋もれている」表向きカードまで組札への移動候補に含めていた。
- **影響**: 実際には単独で動かせないカードをヒントとして提示し得る。
- **対策**: 組札への候補を「各列の一番上のカード」＋「waste のトップ」に限定。場札への候補（連番スタックの先頭になれる表向きカード）は従来通り維持。Node のロジックテスト 41/41 で回帰確認。

### バグF: 山札クリックで2枚ドローされる（二重発火）
- **現象**: Stock を1回クリックすると2枚ドローされ、Moves が +2 されていた（上級モードでは1クリック6枚）。
- **原因**: Stock 領域の外側 `div` と、その上に重なる内側 `Card` の**両方に `onClick={drawCards}`** が設定されており、クリックイベントがバブリングして 2 回発火していた。
- **対策**: 内側 `Card` の重複した `onClick` を削除（外側 `div` が領域全体のクリックを処理）。ブラウザ実機で「1クリック=1ドロー」を確認。

- README を実装に合わせて更新（コンフェッティは Framer Motion 実装である旨を反映、ライブデモ／ライセンス／導入手順を追記）。
