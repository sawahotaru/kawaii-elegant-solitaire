ソリティアゲーム開発仕様書

1. プロジェクト概要
ゲームタイトル: Kawaii & Elegant Solitaire
ジャンル: トランプゲーム（クロンダイク・ソリティア）
プラットフォーム: Webブラウザ（PC / スマートフォン両対応）
コンセプト: 手軽に美しいデザインで癒やされたいユーザー向けの、可愛らしくエレガントなソリティア。

2. 技術スタック（Technical Stack）
以下の技術を用いて開発を行うこと。
言語・フレームワーク: React, TypeScript
スタイリング: Tailwind CSS
アニメーション: Framer Motion（カード移動、クリア演出）
ドラッグ操作: dnd-kit または React DnD
ビルド・デプロイ: Vite等を使用し、一般的なレンタルサーバーに配置可能な静的ファイル（HTML/CSS/JS）として出力する。

3. ゲームルール・ロジック

3.1 基本ルール
標準的なクロンダイク・ソリティアのルールに準拠する。
山札 (Stock): 補充用カードの山。
場札 (Tableau): 7列のカード。赤と黒が交互になるように降順（K→A）で重ねられる。
組札 (Foundation): マークごとにAからKまで昇順に重ねる場所。4箇所。

3.2 難易度設定（モード）
ゲーム開始時およびメニューから以下の3モードを選択可能にする。
項目	入門レベル	通常レベル	上級者レベル
ドロー枚数	1枚ずつ	1枚ずつ	3枚ずつ
Undo (やり直し)	無制限	最大10回	最大3回
Hint (ヒント)	無制限	最大3回	最大1回
スコア倍率	1.0倍	1.5倍	2.5倍

3.3 機能詳細
やり直し (Undo):
ゲームの状態（State）をスタックで管理し、1手前の状態に戻す。
モードごとの回数制限を実装する。
ヒント (Hint):
以下の優先順位で移動可能なカードを探索し、視覚的に強調表示（ハイライト等）する。
場札から組札へ送れるカード
場札の間で移動可能なカード
山札から移動可能なカード
該当がない場合はトースト通知等で「動かせるカードがありません」と表示。
スコア・タイマー:
プレイ時間を計測するタイマー（カウントアップ）。
カード移動やクリアに応じたスコア加算（詳細は標準的なルールに準じ、最後に倍率を掛ける）。

4. UI/UX デザイン仕様

4.1 画面構成（レイアウト）
レスポンシブ対応とし、PCとスマホで見やすさを維持すること。
ヘッダー（上部）:
スコア表示
タイマー
残りUndo回数 / 残りHint回数
プレイエリア（中央）:
組札エリア（4枠）
山札エリア（ドロー前・ドロー後）
場札エリア（7列）
フッター/メニュー（下部またはオーバーレイ）:
リスタートボタン
モード変更（New Game）
設定（音量等の予備枠）

4.2 操作・インタラクション
ドラッグ＆ドロップ: カードを掴んで移動させる直感的な操作。
タップ/クリック移動: 移動可能なカードをダブルクリック（またはタップ）した際、適切な場所（組札など）へ自動移動させる。
アニメーション:
カード配布時のディールアニメーション。
カード移動時の滑らかな補間（Framer Motion使用）。
クリア時の「ファンファーレ・エフェクト」（カードが弾ける、紙吹雪が舞うなどの祝福演出）。

4.3 デザイン素材（仮実装の方針）
開発初期段階では、画像素材の代わりにCSS（Tailwind）による色分けと文字表示でプレースホルダーを作成すること。
プレースホルダー要件:
カード裏面: パステルカラーの単色またはCSSパターン。
カード表面: 白背景に数字とマーク（赤/黒）を視認性高く配置。
背景: 落ち着いたグラデーション（例: ベージュ系、薄いピンク系）。
※ 正式な画像素材（AI生成予定の「花と宝石」モチーフの絵札など）は後日差し替えるため、画像パスを定数管理するなど差し替えやすい構造にしておくこと。
